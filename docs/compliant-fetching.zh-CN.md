# 合规抓取

`feed-all` 采用保守的 RSS-first 访问策略。

## 规则

- 优先使用 RSS 或 Atom，而不是抓取 HTML。
- 使用稳定且可识别的 User-Agent。
- 遵守 robots.txt、站点条款、访问控制和频率限制。
- 使用条件请求，并在实现支持时遵守 `Retry-After`。
- 重新验证重定向目标，保留响应大小和 SSRF 限制。
- 遇到 CAPTCHA、登录、付费墙或反爬挑战页时停止。
- 将 403 和 401 视为授权失败，不将其当作绕过访问控制的提示。
- 429 和 503 只能根据服务器指示有限重试。

robots 规则是爬虫信号，不是访问授权的替代品。法律和合同义务取决于来源
和使用场景；本文是工程策略，不构成法律意见。

## 来源健康

Agent 应保留来源 URL、最后状态、错误原因以及是否被人工禁用。来源失败时应
诊断或替换，而不是反复请求。

## 不在范围内

隐身爬取、破解 CAPTCHA、复用凭证、为规避限制而轮换代理，以及未经授权抓取
私有或付费内容。
