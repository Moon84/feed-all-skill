# 数据生命周期

## 默认持久化

服务保存订阅和条目元数据，包括来源 URL、标题、来源名称、发布时间、作者、
有上限的 preview、阅读状态和健康状态。默认数据目录为 `~/.feed-all`，可用
`FEED_ALL_DATA_DIR` 修改。

## 当前 0.2.x 行为

已发布服务的 `rss_get_entry(fetch_content=true)` 可能抓取并持久化链接正文。
这是当前行为，不是目标保留策略。

## 规划中的链接优先策略

规划策略保存链接、有上限的 preview、hash 和状态。正文、图片、附件和外部
搜索正文默认只作为临时内容，除非用户明确要求上层应用持久化。

规划配置包括 `FEED_ALL_MAX_ENTRIES`、`FEED_ALL_RETENTION_DAYS` 和
`FEED_ALL_MAX_PREVIEW_CHARS`。它们是设计输入，不是已发布 0.2.x 的环境变量。

## 生命周期

```text
发现 -> 订阅 -> 保存元数据 -> 被选中阅读
     -> 临时读取正文 -> 用户确认后可选持久化
```

不能因为内容出现在订阅中，就静默复制到更大的知识库。
