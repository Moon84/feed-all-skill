# 架构设计

## 定位

`feed-all` 是 Agent 信息跟踪系统中的来源和元数据层，不是爬虫集群、定时
任务系统、新闻编辑器或全文知识库。

## 系统上下文

```text
MCP Host（Codex / Claude / 其他）
  -> feed-all MCP 服务
      -> 订阅管理
      -> 刷新协调
      -> 合规抓取层
      -> 元数据存储
      -> 保留策略（规划中）

Agent
  -> 筛选 -> 总结 -> 提醒 -> 请求选中正文

用户 / EntroFeed
  -> 确认订阅 -> 更新偏好 -> 保存选中产物
```

## 职责边界

| 模块 | 职责 |
| --- | --- |
| feed-all | 来源、订阅、刷新、元数据、链接、阅读状态 |
| Agent | 相关性、事件去重、总结、提醒 |
| 用户 | 确认订阅、保存和偏好修改 |
| EntroFeed | 长期知识、分析产物、本体和用户画像 |

## 数据流

```text
发现 -> 验证 -> 订阅 -> 刷新 -> 保存元数据
                            -> Agent 筛选候选
                            -> 按需读取正文
                            -> 可选交给上层持久化
```

服务没有内置周期性 scheduler 或 Digest。Host Agent 可以在用户配置的定时
任务中调用刷新。

## 版本边界

本文档描述已发布的 0.2.x 契约。条件 HTTP、容量保留、外部发现和新增存储
工具属于规划能力，不能写成当前工具。
